## What Works

- **2026-07-07** ‚Äî "Resource not generated yet" tri-state: catch the 404 in the fetcher (`ApiError.status === 404` ‚Üí return `null`) so TanStack Query lands in the success state with `data === null` ‚Äî the page then branches cleanly: `null` ‚Üí first-visit CTA, data ‚Üí content, `isError` ‚Üí genuine failure only. No error-object probing in components. Evidence: `client/src/lib/api.ts` (`fetchOnboarding`), `client/src/lib/hooks/onboarding.ts`.

- **2026-06-14** ‚Äî `formatCost` (`src/lib/cost.ts`) distinguishes MISSING data (`null`/`undefined` ‚Üí "‚Äî") from a genuine zero (`0` ‚Üí "$0.00"), widens precision for sub-cent values (~2 sig figs), and trims trailing zeros to a 2dp floor ("$0.06" not "$0.060", "$0.0013" not "$0.00"). Reuse it for any per-run money display.
- **2026-07-13** ‚Äî Testing a Recharts `<Tooltip content={...}>` by simulating real mouse hover in jsdom is unreliable: `ResponsiveContainer` reports 0√ó0 dimensions under jsdom (visible as a console warning on every chart render in this test suite already), so Recharts' internal active-index/coordinate math has no real geometry to key off, making `fireEvent.mouseOver`/`mouseMove` on the SVG a flaky way to trigger the tooltip. Pattern that works: export the tooltip's content-renderer as its own named component (e.g. `ChartTooltip({active, payload})`) and unit-test IT directly via RTL `render()` with hand-built `{active, payload}` props ‚Äî bypasses the mouse-simulation problem entirely, deterministic, and still exercises the real formatting logic. Evidence: `client/src/vendor/ui/charts/LineChart.tsx` (`ChartTooltip`), `client/src/vendor/ui/charts/LineChart.test.tsx`.
- **2026-07-17** ‚Äî Before building a new radial/ring gauge for a "small ring badge with a number" design (e.g. a corner badge on a summary card), check `@devdigest/ui`'s `CircularScore` primitive first (`score: number` 0-100, optional `size`/`stroke`, built-in green/amber/red threshold coloring via `--ok`/`--warn`/`--crit`) ‚Äî it already covers that exact shape and is a one-line import (`import { CircularScore } from "@devdigest/ui"`), no new SVG needed. Used to fix the Agent Performance dashboard's "AVG ACCEPT RATE" card to match the design mock (`size=36`, badge positioned `absolute top/right` inside a `position:relative` card). Note this means the Stats tab's `AcceptRateGauge` (a bespoke full SVG ring built for the stats-tab-enrichment feature, `size=80`, own null-state) duplicates the same visual shape as `CircularScore` ‚Äî worth consolidating onto the shared primitive next time that component is touched, rather than maintaining two ring-gauge implementations. Evidence: `client/src/vendor/ui/primitives/CircularScore.tsx`, `client/src/app/agent-performance/_components/SummaryCards.tsx`, `client/src/app/agents/[id]/_components/AgentEditor/_components/StatsTab/_components/AcceptRateGauge.tsx`.

## What Doesn't Work

- **2026-07-11** ‚Äî `@devdigest/ui`'s `charts/Sparkline.tsx` (and `MetricCard`, which renders one internally whenever a `trend` prop is passed) divides by `data.length - 1` to place points ‚Äî a trend array of length 1 produces `0/0 = NaN`, which React then throws into `<path d="MNaN,18.0">` / `<circle cx="NaN">` at render, spamming console errors (caught live via Playwright screenshot verification, not by the component test ‚Äî jsdom doesn't surface SVG attribute-parse errors as failures). Any caller passing a real (not >=2-length-guaranteed) trend array MUST gate it: `trend={points.length >= 2 ? points : undefined}`. `AgentEvalCard`'s own local Sparkline already guarded this way; `MetricCard` usages must do the same manually since the vendor component doesn't guard internally. Evidence: `client/src/vendor/ui/charts/Sparkline.tsx`, `client/src/app/eval/[agentId]/_components/AgentEvalDetailView.tsx`.
- **2026-07-11** ‚Äî A client-side "date range" filter (e.g. last 7/30/90 days) that defaults to a narrow window computed against real `Date.now()` can silently render an empty state on first load whenever the underlying data (seed/demo data, or just an agent that hasn't run recently) is older than the window ‚Äî there's no visible cue that a filter, not an empty dataset, is hiding everything. Default such filters to "All time" and let the user narrow deliberately. Evidence: `client/src/app/eval/[agentId]/_components/AgentEvalDetailView.tsx` (caught before shipping ‚Äî a 30-day default against 2026-07-11 "now" hid May-2026 seed batches entirely).

- **2026-07-11** ‚Äî `fetchEvalDashboard()` called with no `agentId` (the null-owner/workspace variant) returns `recent_runs` as bare `EvalRunRecord[]` ‚Äî per-*case* rows with no agent name/id attached. It cannot power a workspace-wide table that needs to show which agent a row belongs to (e.g. an "all agents" recent-runs view). Fix: drop the null-owner call and instead fan `fetchEvalBatches(agentId)` out across every agent via `useQueries`, tagging each returned `EvalRunBatch` with the agent's own name/id client-side, then merge+sort by `ran_at`. `fetchEvalBatches` is already agent-scoped and already used by `EvalsTab`, so this reuses a proven endpoint instead of extending the dashboard contract. Evidence: `client/src/app/eval/_components/EvalDashboardView.tsx`, `client/src/lib/api.ts:195` (`fetchEvalBatches`).
- **2026-07-07 (later)** ‚Äî An inline preview panel appended AFTER a long list inside a narrow column (`maxWidth:580`) opens below the fold and reads as "the button does nothing" ‚Äî the Context tabs' original `<pre>` preview was state-wired correctly yet reported broken for exactly this reason. For any on-demand content preview use the vendor kit `Drawer` (`@devdigest/ui`, `position:fixed; inset:0` right-side overlay ‚Äî RunTraceDrawer precedent): visibility becomes layout-independent. Also supersedes the `SafeMarkdown` evidence path above: the component was promoted to `client/src/components/SafeMarkdown/` (old `_components/` copy deleted). Evidence: `client/src/app/agents/[id]/_components/AgentEditor/_components/ContextTab/ContextTab.tsx`.
- **2026-07-07** ‚Äî `vendor/ui/primitives/Markdown` is NOT safe for untrusted (repo-controlled) content: it renders `<a href={href}>` without filtering `javascript:`/`data:` URLs. For untrusted markdown use the `SafeMarkdown` wrapper pattern (custom `a` component with an `isSafeHref` check, no `rehype-raw` so inline HTML stays inert). Evidence: `client/src/vendor/ui/primitives/Markdown.tsx:53-57`, `client/src/app/project-context/_components/SafeMarkdown.tsx`.
- **2026-07-15** ‚Äî Despite the rule directly above, `FindingCard.tsx` (the single-agent PR page's finding detail view ‚Äî Accept/Dismiss/Turn-into-eval-case) had shipped rendering `f.rationale`/`f.suggestion` (untrusted LLM output) through the raw `Markdown` primitive, not `SafeMarkdown` ‚Äî found only by diffing it against a hand-rolled multi-agent duplicate while reusing this component elsewhere. A documented rule doesn't guarantee every existing call site follows it ‚Äî when reusing/auditing a component that renders agent/LLM text, grep its own source for `<Markdown>` before trusting it's already safe. Fixed. Evidence: `client/src/app/repos/[repoId]/pulls/[number]/_components/FindingCard/FindingCard.tsx`.
- **2026-07-15** ‚Äî `useRunEvents(runIds)`'s `running` return value is the AGGREGATE state across the whole batch ‚Äî it only flips to `false` once every EventSource in the list has closed (`open <= 0` in the hook). Gating a query-invalidate effect on that flag (`if (!running && wasRunningRef.current) invalidate()`) means the invalidate fires only once when the SLOWEST run in the batch finishes ‚Äî an agent that completed early in a multi-agent fan-out never showed its score/findings until every sibling was also done. Fix: track per-run terminal SSE events (`kind === "result" | "error"`) in a `Set<runId>` ref and invalidate the instant a NEW one appears, regardless of whether `running` is still `true` for the batch as a whole. Also don't forget the SECOND query that needs invalidating: Tabs mode's finding cards read `usePrReviews` (`["reviews", prId]`), a completely separate query from `["multi-agent-run", runId]` ‚Äî invalidating only the latter left findings stuck until a full page reload even after this fix. Evidence: `client/src/app/multi-agent/[runId]/_components/MultiAgentResultsView.tsx`, `client/src/lib/hooks/reviews.ts` (`useRunEvents`).

## Codebase Patterns

- **2026-07-15** ‚Äî A new top-level route's `page.tsx` renders with NO sidebar/topbar at all if its client view forgets to wrap itself in `<AppShell>` ‚Äî `app/layout.tsx` (the Next.js root layout) does NOT provide the shell; every leaf page opts in individually, in every return branch (loading/error/success), not just the happy path. Caught via screenshot: `multi-agent/[runId]/_components/MultiAgentResultsView.tsx` shipped without the wrap while its sibling `multi-agent/configure/_components/ConfigureRunView.tsx` had it correctly. A page missing the wrap also silently loses the max-width the sidebar's presence normally forces: cards using `flex:1` with no `maxWidth` container stretch to the full viewport width, reading as "broken" layout once real (non-empty) content is present ‚Äî cap such grids with e.g. `gridTemplateColumns: "repeat(auto-fill, minmax(240px, 300px))"` + `justifyContent: "flex-start"` inside a `maxWidth`-bounded wrapper, not bare `flex:1`. Evidence: `client/src/app/multi-agent/[runId]/_components/MultiAgentResultsView.tsx`, `client/src/app/multi-agent/configure/_components/ConfigureRunView/ConfigureRunView.tsx`.

- **2026-07-15** ‚Äî The standalone design HTML exports under `/Users/helga/Sites/neoversity/lessons/DevDigest Design (standalone) (N).html` do NOT contain component source as plain text ‚Äî `grep`-ing for a component/string name (e.g. `ConflictsSection`, `"Where agents disagree"`) finds nothing even though the component is real and mounted. The file stores each script as a gzip+base64 blob in a big JSON map (`"<uuid>":{"mime":"application/javascript","compressed":true,"data":"<base64>"}`), referenced by `<script type="text/babel" src="<uuid>">` where the "src" is a lookup key, not a URL. One blob per top-level design module (~46 blobs in a 1.7MB single-line file); the one containing a target component usually contains its whole feature slice (e.g. one blob had `ScreenMultiAgent` + `ConflictsSection` + `RunConfig` + `ColumnsView` + `TabsView` together).
